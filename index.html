Definição de servidor

No contexto das redes de computadores, os servidores (ou servers) ''servem'' os usuários (também chamados de clientes) com informações requisitadas através de protocolos variados (HTTP1 (Hypertext Transfer Protocol), FTP2 (File Transfer Protocol), ...). Por exemplo, poderíamos como clientes de um aplicativo bancário requisitar de um servidor o saldo de nossa conta bancária, que seria retornado para nós caso nossa solicitação fosse validada (através de senhas, por exemplo).

Essa separação (chamada de modelo cliente-servidor) é vantajosa pois permite uma melhor performance e maior especialização em ambos os lados da solicitação e, além disso, maior segurança, pois isola os elementos que cada usuário deve ter acesso. Claramente, não devemos ter acesso a informação do saldo bancário de outros clientes; portanto, elas ficam armazenadas em um servidor que decide quem pode a acessar através de autenticação e validação. Caso essa informação estivesse em cada computador de cliente, seria facilmente acessível por truques de leitura de memória. Tendo isso em mente, a importância dos servidores é clara; e, para fins de ilustração, segue uma citação interessante sobre eles:

"Servers don’t sleep; they’re the unsung heroes of the internet, ensuring data flows around the clock."

Funcionamento de um servidor

A característica principal de um servidor é aguardar a solicitação de um cliente, atendendo a ela (caso seja considerada válida) e retornando para ele as informações desejadas. Para interagir com os clientes, é necessário uma interface de usuário que possa enviar a solicitação e de uma rede de comunicação existente entre o cliente e o servidor (geralmente a Internet).

Para que os servidores possam processar essas solicitações, utiliza-se software como Apache, nginx ou outros especializados em diferentes protocolos. Esse software processa as solicitações (no caso de Apache, solicitações HTTP) e instrui ao resto do servidor o que fazer, de acordo com as aplicações instaladas nele. Por exemplo, o Apache poderia processar uma solicitação HTTP requisitando dados bancários do cliente XY e, através disso, enviar uma mensagem para que o servidor retire do banco de dados da aplicação essa informação.

Quanto a sistema operacional, podem-se usar vários diferentes como Linux (Fedora, CentOS, Ubuntu...), Windows Server, FreeBSD, entre outros. O hardware depende nas necessidades da aplicação, por exemplo, uma aplicação que necessita de grandes cálculos complexos a cada solicitação necessita de um hardware muito mais potente que um simples aplicativo de blog.

Hospedando servidores

Para hospedar um servidor, pode-se fazer por conta própria através da montagem por conta própria dos seus componentes de hardware e software (self-hosting) ou, de uma maneira mais prática, contratar uma empresa para que esta realize a hospedagem para você, com diversos benefícios possíveis como ser multiplataforma, vir com o registro de domínio de forma conjunta, etc.

Todo website vai necessitar de um nome de domínio, que deve ser registrado no local correto (Registro BR para sites .br), que deve ser pago anualmente e será mapeado para um endereço IP fornecido pelo seu serviço de hospedagem ou por você mesmo, caso realize self-hosting.

Agora, veremos alguns exemplos de servidores. O primeiro deles é o servidor web, que hospeda os websites que você acessa diariamente. Este é responsável por, ao ser solicitado pelo cliente o website através do seu nome de domínio (que, por vez, é mapeado para um endereço IP), retornar arquivos HTML, CSS e JavaScript para serem executados pelo navegador e permitir o acesso ao website. Eles também gerenciam as informações que o website necessita para funcionar, comunicando-se com um banco de dados.
